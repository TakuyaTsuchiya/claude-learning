## 講座全体の教育方針アイデア（土屋メモ）

### 「ローカル便利ツール量産」アプローチ
- 土屋さん自身がClaude Codeを習得した方法：Webにデプロイせず、社内向け便利ツールをローカルでいっぱい作りまくって覚えた
- メリット：
  - セキュリティをそんなに気にしなくていい（ローカル完結）
  - 手を動かして先に完成形を作り、後から仕組みを学ぶ流れが効果的
  - 非エンジニアでも「自分の業務に使えるもの」を作れる実感が得られる

### 魔法体験のお題案：「個人情報マスキングツール」
- テキストを貼り付けると、名前・電話番号・メールアドレス・年収などを自動検出して伏せ字にするローカルWebアプリ
- プロンプト例：「個人情報をマスキングするWebアプリを作ってください。テキストを貼り付けると、名前・電話番号・メールアドレス・年収などの個人情報を自動で検出して伏せ字にしてくれるものです。」
- 良い点：
  - 「AIに個人情報を入れたくない」→ だからローカルで動くClaude Codeで作る、という動機が明確
  - TODOアプリと違ってやりつくされていない
  - 営業・事務系ペルソナ（B）にも刺さる実務ユースケース
  - HTML+JSだけで動く、環境依存なし
- Boris Chernyの「箱から出したままでも十分使える」思想とも合致（docs/boris-quotes.md参照）

### 必ず触れたいポイント：トークン管理とスクリーンショット活用

**トークン管理が重要な理由：**
- トークンが増える → 参照情報が多くなる → コンテキストウィンドウが溢れる → AIが迷う（情報過多）
- だからこまめに/clearすることが重要
- トークン = シンプルに言えば文字数。テキスト < 画像 < 動画の順に消費量が増える
- 節約の意識を早い段階で持たせる

**スクリーンショット活用の教え方：**
- Claude Codeはスクリーンショットを読める
- アプリを動かす → スクショを撮る → Claude Codeに見せてフィードバックをもらう
- このサイクルで開発が加速する
- 非エンジニアでも「見せて直してもらう」ができるのが強い

**講座での配置案：**
- S1の基本コマンド動画（V5）で/clear、/cost、/compactを教える際にトークンの概念を導入
- 魔法体験（V6）でスクショ活用を実演：作ったアプリのスクショを撮ってClaude Codeに改善させる

### この講座の差別化ポイント：「動画で具体的な動きを見せる」

- ネット上の教材は多いが、テキストベースがほとんど。この講座は**動画**
- 概念を長々説明するより、**具体的な操作画面を見せる**方が圧倒的にわかりやすい
- 例：GitHubは概念（ブランチ、マージ、プルリクエスト…）で説明すると複雑に聞こえるが、実際にやってるところを見れば**「単純にセーブポイントを作るだけ」**
- 土屋さん自身が初心者時代に感じた「具体的な動きを見たかった」という体験がベース
- **動画だからできること**：画面収録でClaude Codeが動く様子をリアルタイムで見せる

**講座全体の方針：**
- 概念説明は最小限に → まず動いてるところを見せる → 後から「今のは何だったか」を解説
- これはS1の「先に完成形を作って後から仕組みを学ぶ」アプローチと一貫
