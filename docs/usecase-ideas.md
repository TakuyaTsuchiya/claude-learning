## 講座方針：会議資料はスライドではなくHTMLで作れ

スライドがダメな理由：
- AIが生成してもレイアウトの微調整が地獄
- 1ページの情報量が少なく枚数が膨れる
- 静的で、データの絞り込みやインタラクションができない

HTMLが正解な理由：
- Claude Codeに一言で生成させられる
- 自由なレイアウト、情報量の制限なし
- インタラクティブ（グラフ、フィルター、折りたたみ、検索）
- URLを送るだけで共有完了
- この講座のレビューサイト自体がその証明

→ S3のスキル実演 or S4のユースケースで「会議資料HTML化」を入れるべき

## 本質：BPOの巻き取り

つまりこの講座が育てるのは「Claude Code × MCPで業務システムに直接つなぎ、BPOが人手でやっていた業務を全部自動化できる人材」。

BPOの典型業務：
- 業務システムにログイン→データ抽出→Excel加工→レポート作成→納品
- これが「MCPで接続→Claude Codeに指示→自動処理→PDF/Excel出力」に置き換わる

受講生がこの構造を理解すれば、どの業界のどの業務でも同じパターンで巻き取れる。

## 講座の核心メッセージ

**「速度が全て。細部に時間をかけるな、動くものを先に見せろ。」**

ビジネスで勝つのは完璧な資料を作る人じゃない。
顧客が欲しいものを最速でプロトタイプとしてポンと見せられる人。
- 見積もり → その場で作って送る
- 会議資料 → 会議の10分前にHTML生成
- 提案書 → 商談中にプロトタイプ見せる
- 議事録 → 会議終了と同時に共有
- メール → 考える前に出す

Claude Codeが変えるのは品質ではなく速度。速度が打席数を増やし、成果につながる。

**「動き出すエネルギーをゼロにする」**

人間の最大のボトルネックは「何をやるか決めて、最初の一歩を踏み出す」こと。
動き出したらなんてことないのに、そこまでが重い。
AIが着火剤になる：
- 何をやるか整理してくれる（Slack・Calendar・メールから自動抽出）
- 最初の一歩を用意してくれる（テンプレート作成済み、あとは埋めるだけ）
- 「Enter押すだけ」の状態まで持っていく

**「リサーチ時間をゼロにする」**

どの業種でも仕事の大半は「調べる時間」に消えている。
- 営業 → 競合調査、顧客情報収集
- 法務 → 判例調査、契約条文の確認
- 経営者 → 市場調査、業界動向
- 投資家 → 銘柄分析、決算読み込み
- 経理 → 税制改正、勘定科目の判断
- 人事 → 候補者の経歴確認、制度調査

Claude Codeは「調べて→まとめて→判断材料にする」を一気通貫で回す。
人間は判断だけに集中できる。
