# Claude Code講座 カリキュラム設計書

## 講座の位置づけ

1. **AIスクール後半のステップアップ専門講座**: ブラウザベースのAIツール（ChatGPTやGemini等）の基本操作に慣れた受講生が、次のステップとして選ぶ実践的な専門講座
2. **自律型AIエージェントへの橋渡し**: 「指示を出して回答をもらう」受け身のAI利用から、「AIにPCを操作させ、自律的に作業を代行させる」エージェンティックなAI利用への登竜門
3. **非エンジニア向けのDX・自動化スキル習得の場**: エンジニア向けの「開発ツール」としてだけでなく、営業やバックオフィスなどの非エンジニアが自らの業務を自動化・効率化するための「最強のビジネスツール」

## 講座の目的

1. Claude Codeの自力での導入と基本操作の習得（CLI恐怖心の克服）
2. プロの「AI駆動開発フロー」の再現（スペック駆動開発、CLAUDE.md活用）
3. アイデアを形にし、公開するスキルの獲得（Vercelデプロイ）
4. 実務での業務改善とマネタイズの実現（DX、副業・案件獲得）

## 講座の狙い

1. **「魔法体験」によるモチベーション最大化**: 最序盤で「AIが自動でアプリを作り上げる」という成功体験を提供
2. **認知負荷を下げるカリキュラム設計**: GitHubやバージョン管理を本編（入門）から外し、挫折率を極限まで下げる
3. **「Vibe Coding」からの脱却**: スペック駆動開発で実務レベルのスキルを養成
4. **「自分ゴト化」させるユースケース**: スクレイピング、請求書処理、社内Q&Aなど実務に直結する事例

## 制作パイプライン

既存の教材開発マニュアルに準拠（7ステップ）:

1. **構成作成** → 進捗管理シート（本ドキュメントで完了）
2. **台本作成** → NotebookLM/GEMS + 台本作成プロンプト → スプレッドシート
3. **資料（スライド）作成** → 下記「スライド生成ワークフロー」参照
4. **スライド完成** → Figmaデザイン調整・レビュー → 下記「STEP 4 Figmaワークフロー」参照
5. **撮影・動画編集** → HeyGenリップシンク等
6. **クイズ・試験問題作成** → NotebookLM + クイズ作成プロンプト
7. **完成** → WisdomBase LMSへ登録（/wb-* スキル群で自動化）

### STEP 3 スライド生成ワークフロー（詳細）

```
台本xlsx の「スライド種別・名称・用途」（Row 13〜）
  ↓ コピー
[STEP 3.1-3.2] Gems（Gemini）でYAML形式に変換
  資料作成/Nanobanana（最新）
  https://gemini.google.com/gem/1VCJwTxC8jd_6cvYjRg9mIHlYfU8fR7g0?usp=sharing
  ↓ 生成されたYAMLをコピー
[STEP 3.3] NotebookLM「スライド」→「カスタムプロンプト」にYAMLペースト → 生成
  ↓ スライド資料ダウンロード
[STEP 3.4] ダウンロードした資料と台本の「スライド種別・名称・用途」にズレがないか確認
  ↓
[STEP 3.5] 資料ホルダーの格納庫に保存
```

| ステップ | やること | 使うツール |
|---------|---------|-----------|
| 3.1 | 台本xlsxからスライド情報（種別・名称・用途）をコピー | 台本スプレッドシート |
| 3.2 | GemsでYAML形式に変換 | 資料作成/Nanobanana Gems |
| 3.3 | NotebookLMでスライド資料を生成 | NotebookLM（カスタムプロンプト） |
| 3.4 | ダウンロードして台本との整合性を確認 | 目視確認 |
| 3.5 | 資料格納庫に保存 | Google Drive |

### STEP 4 Figmaデザイン調整ワークフロー（詳細）

```
NotebookLMで生成されたスライド資料
  ↓
Figma「AI School Slide」プロジェクトに配置
  ↓
デザインチーム（WEN Design Account等）がレビュー・コメント
  ↓
赤背景 = 修正対象スライドとして識別
  ↓
微修正・デザイン調整
  ↓
完成 → 撮影・動画編集へ
```

**Figmaプロジェクト構成:**
- プロジェクト名: AI School Slide
- ページ単位でモジュールを管理（Module 3, Module 4, Module 5...）
- 既作テンプレートが存在し、統一されたデザインを適用

**命名規則:**
- レイヤー・フレーム名: `CC-S{セクション}-V{動画}-Page{ページ}`
- 例: `CC-S1-V1-Page1`（セクション1・動画1・1ページ目）
- 参考: M5では `m5-s4-v5-p8` のような形式で管理されていた

**レビューフロー:**
- Figmaコメント機能でデザインチームからFBが入る
- 赤背景のスライド = レビュー中・修正対象
- コメント解決 → 完成
