# BACKSTAGE社 ヒアリングアジェンダ（S6-V4用）

> 及川さん（BACKSTAGE社エンジニアチームリード）への30分ヒアリングのアジェンダ。
> 目的：S6-V4「現場のClaude Code活用」動画の素材となるネタ出し。

---

## 背景・前提
- S6ラスト2本（V4・V5）でBACKSTAGE社の実例を紹介したい
- **先方負担を極小にする設計**：
  - Zoom 30分の雑談ヒアリングのみ
  - 画面収録・編集は土屋側で全部実施
  - 出演（顔出し・声出し）は任意。出なくても成立する構成
- 受講者の9割は非エンジニア。「同じ使い方をすぐ真似できる粒度」を重視
- BACKSTAGEのファンが多いため、「現場で使われている」事実そのものが価値
- 社内でClaude Codeを使っているのは及川さんたちエンジニアのみ（非エンジニアは未使用）

## ゴール
- 映像化できる「ネタ」を5〜8個引き出す
- そのうち非エンジニア受講者が真似できる粒度のものを3〜5個選ぶ

---

## アジェンダ（6項目・30分）

### 1. 便利だった使い方ベスト3〜5（個人技）
- 最近「これ助かった」と感じたClaude Codeの使い方
- 地味だけど毎日使うやつ優先

### 2. 使ってるSkill
- 自作Skill／公式Skill／おすすめSkill
- チーム配布してるSkillがあれば特に

### 3. 使ってるMCPサーバー
- 実務で常時繋いでいるもの
- 意外と便利だったMCP

### 4. 日常ワークフロー
- 朝〜夜でClaude Codeがどこに挟まるか
- 他ツール（Slack / GitHub / Linear 等）との連携点

### 5. 使わなくなった使い方・ハマった話
- 期待してたけど手放したもの
- リアルさが出るのでむしろ欲しい

### 6. チーム運用の観点
- チーム共通Skillを配布している？
- MCP設定は標準化している？
- コードレビューにClaude Codeを絡めている？
- 新人エンジニアのオンボーディングでの活用
- 「これはチーム全員がやってる」習慣

---

## 収録・編集方針（こちら側）
- ヒアリング後、土屋が自分の環境で再現して画面収録
- 映像に及川さん本人が出るかは先方の意向優先
- 「BACKSTAGE社の◯◯さんが実際に使っているテクニック」というテロップで繋ぐ
- 機密にあたるものは除外／抽象化する

---

## V4／V5の切り分け方針（6分×2本構成）

ヒアリングは30分1回で両方分まかなう。アジェンダ（上記6項目）はそのまま共用。

### V4：個人の道具箱（6分）
- 便利な使い方ベスト3〜5 から 2〜3個
- 使っているSkill紹介
- 使っているMCP紹介
- **「非エンジニアでも真似できる」粒度を優先**
- 狙い：即効性の満足感。見終わってすぐ真似できる

### V5：チーム運用＋講座の締めくくり（6分）
- チーム共通Skill・MCP標準化・レビュー文化・オンボーディング
- ハマった話・使わなくなった話（リアル感）
- **ラスト1分は土屋さんの締めメッセージ**：「皆さんが作ったプロトタイプの先には、こういう現場がある」
- 狙い：未来ビジョン。講座全体の余韻＆次のステップ示唆

### 分け方の根拠
- V4（道具）→ 真似しやすい、即効性
- V5（チーム＋締め）→ 未来ビジョン、余韻
- V4 → V5 で「個 → 組織」の流れ、受講者の次のステップを自然に示せる
