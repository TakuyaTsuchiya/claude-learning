【BACKSTAGE社独自のCLAUDE.mdテンプレート大公開】

CLAUDE.mdといえば、一般的にはこんな形が多いです：
```
# プロジェクト名 — Claude Code設定

## 技術スタック
- Next.js 16 + React 19 + TypeScript
- Tailwind CSS 4 + shadcn/ui
- Supabase（DB・認証）
- Vercel（ホスティング）

## コード品質
- TypeScript: any型はMUST NOT。unknown + 型ガードを使う
- 1関数50行以下
- 既存パターンを踏襲。新しいパターンを勝手に導入しない

## ワークフロー
- 変更前に既存コードを読んで既存パターンに従う
- 変更後はMUST `pnpm build` でビルド確認
```

もしくは Boris Cherny（Claude Code開発責任者）式の6原則＋`.claude/`全体設計：
・Plan Mode Default（3ステップ以上は計画を先に）
・Self-Improvement Loop（ミス記録を次セッションで読み返す）
・Verification Before Done（完了前に「スタッフエンジニア視点」で自問）
・Subagent Strategy（リサーチはサブエージェントに投げる）
・Demand Elegance（設計判断は複数アプローチ比較）
・Autonomous Bug Fixing（バグはまず自分で調査）
＋ `.claude/` を CLAUDE.md / settings.json / commands/（Skills推奨）/ rules/ に分離
