# S2-V6 新設ドラフト：Claude Codeを安全に使う基本を押さえよう

**位置**：S2の最後（V6として新設、S2は5本→6本に）
**由来**：旧S5-V2「最も重要なセキュリティの考え方」をS2に移動・内容変更。環境変数／APIキーの話は削除し、S2入門セクション卒業前に押さえるべき「安全に使う基本」に絞る
**タイトル**：Claude Codeを安全に使う基本を押さえよう（22字、35字以内OK）

---

## 構成7項目

### ゴール
権限3段階を使い分け、承認画面を読んでCtrl+Eで解説を確認し、rm -rf 等の破壊的コマンドを回避して、安全にClaude Codeを使えるようになる

### 重要トピック
- **権限3段階**（デフォルト／Auto-accept edits／フルオート）の使い分け
  - 日常の橋渡し：スマホアプリの「カメラへのアクセスを許可しますか？」と同じ仕組み
- **承認画面の読み方**：何を聞かれているかを確認／Ctrl+Eで解説を表示して意図を理解する癖
  - 日常の橋渡し：Return連打で全承認するのは、利用規約を読まずに「同意する」を押す癖と同じ
- **破壊的コマンド**：rm -rf 等（AIが意図せず大規模削除を提案するケース）
- **プロンプトインジェクション**：fetch系で外部URLを取得すると、そのページに仕込まれた命令が実行される可能性
- **フルオート権限の使いどころ**：サンドボックス環境（S2-V4既出のポリシー13）なら安全に使える
- **利用金額の上限設定**：意図せぬ多額課金を防ぐ

### 受講者のつまずき
- Return連打で全承認してしまう癖（→承認画面を読む習慣を身につける／Ctrl+Eで意図を確認）
- 「自分は注意してるから大丈夫」と思いがち（→AIが意図せず破壊的提案をする実例を見せ、本人の注意深さとは別次元の事故があると体感してもらう）
- フルオートを怖がって絶対使わない（→サンドボックス前提なら安全、という使い分けを学ぶ）

### 画面操作方針
- スライドは表紙＋権限3段階比較図のみ
- 画面収録で実演：権限モード切替 → 承認画面でCtrl+E → **ダミーディレクトリで破壊的コマンドの挙動を見せる**（実ファイルは破壊しない安全配慮）→ 利用上限設定画面

### 位置付け方針
S2（入門セクション）の締めくくり。V1〜V5で身につけた操作を、**安心して応用フェーズ（S3以降）で使える土台**にする回。次のS3（スキル・MCP活用）やS5（アプリ開発）では権限・承認の判断を自分でできる前提で話が進むため、ここで基礎を固める。環境変数・APIキーの実体験はS5-V5（経費アプリ）に託す

### 参照一覧
- Anthropic公式「Permissions」: https://docs.claude.com/en/docs/claude-code/iam ※**HTML書き込み時にURL有効性を要確認**
- ポリシー13「サンドボックス=安全」（S2-V4既出）
- S6（Supabase接続時に環境変数・APIキー・.envを実体験）

### シーンの流れ（10シーン）
まず冒頭で「Claude Codeは強力な分、使い方次第でファイル消失や情報漏洩に直結します。応用に進む前に、安全に使う基本をここで押さえます」と宣言（50〜60秒） → 権限3段階の全体像（スマホのパーミッション許可と同じ仕組み）→ デフォルトモードの挙動（毎回承認を求める）→ Auto-accept editsの挙動（編集は自動、破壊的操作は聞く）→ フルオートの挙動とサンドボックス前提（S2-V4のポリシー13「サンドボックス=安全」との接続）→ 承認画面の読み方とCtrl+Eの実演 → rm -rf 等の破壊的コマンドの実例（ダミーディレクトリで安全に見せる）→ fetch経由のプロンプトインジェクションの怖さ → Return連打の癖を避けるコツと利用金額の上限設定 → 最後に「環境変数・APIキーの扱いは、S6でSupabaseに接続する場面で実際に扱います」と布石

## 設計の引き出し（使った要素）
- **対比構造**：権限3段階（デフォルト vs Auto-accept vs フルオート）を並べて違いを明示
- **次セクションへの布石**：環境変数の実体験はS5-V5に接続
- **既知題材の再利用**：S2-V4のポリシー13「サンドボックス=安全」を再登場させる
