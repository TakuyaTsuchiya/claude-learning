S1：環境構築と魔法体験

4本 / 20分
入門
目的
Claude Codeをインストールし、自然言語で指示するだけで動くアプリが完成する「魔法」を体感する。箱から出したままでも使えることを実感し、学習意欲に火をつける。
学べる内容
  • V1 講座の全体像とAI駆動開発
  • V2 ターミナルの基本操作（mkdir、cd）
  • V3 Claude Codeのインストールと初期設定
  • V4 自然言語でアプリを自動生成させる体験
1
講座の全体像：Claude Codeがもたらす「AI駆動開発」とは
土屋:ClaudeCodeは「箱から出したままでも使える」と言われているので試してもらいたい。細かい設定をする前に。
2
「黒い画面」の基本操作 — mkdirとcdだけ覚えよう
3
Claude Codeのインストールと初期設定
川島:CLI版インストール後にパスが通らないことがよくある。パスを通す作業も入れると迷子にならない
菅野:公式ドキュメント上はネイティブインストーラーが推奨（npm方式は非推奨）。Windowsの前提条件はGit for Windowsのみ。「Node.js等」は要確認
4
【魔法体験】自然言語で「動くアプリ」を作らせてみよう！

S2：AIの「記憶」と使いこなしの基礎

4本 / 20分
入門
目的
CLAUDE.mdでAIに記憶を持たせ、Planモードで「考えてから動く」習慣を身につける。トークン管理とセキュリティの基本を学び、Claude Codeを安全に使いこなせるようになる。
学べる内容
  • V1 基本コマンドとPlanモード
  • V2 CLAUDE.mdの書き方と階層構造
  • V3 自動メモリ（MEMORY.md）の仕組み
  • V4 セキュリティの基本とサンドボックス
1
基本コマンドとPlanモード — 「考えてから動く」習慣
土屋:auto-compact入る前に/clearしていくのが大事。1つの作業ができたらなるべく/clearする
2
AIの「記憶」を司るCLAUDE.mdの書き方
土屋:ClaudeCode開発者のBoris氏のベスト・プラクティスを紹介したい
菅野:CLAUDE.mdは実運用上はドキュメントのリンク集（インデックス）として活用するのが最適。ルールを書きすぎるとAIの注意力を削ぐし、AIは必ずルールを守るわけではない
3
CLAUDE.mdの階層構造と自動メモリ
4
【重要】セキュリティの基本とサンドボックス
土屋:環境変数を理解してもらいたい
菅野:Return連打は気をつける、rm -rfとか、fetch→プロンプトインジェクションなど。Ctrl+Eで簡単な解説が出る。万が一の利用金額の上限設定も教える
